# マイタスク管理アプリ — SPEC

## 概要

GitHub Issues（`ishiinorifumi-ws/my-tasks`）をバックエンドとした、スマホ縦型のタスク管理Webアプリ。
シンプルな操作で「今やるべきこと」を一目で把握・管理できることを目指す。

---

## ターゲット

- ユーザー: 石井則文（1名専用）
- デバイス: スマートフォン（縦型メイン）、PC でも閲覧可
- 横スクロール禁止

---

## データソース

- リポジトリ: `ishiinorifumi-ws/my-tasks`
- GitHub REST API v3 を使用
- 認証: GitHub Personal Access Token（環境変数でブラウザに保持）

## 機能一覧

### コア機能

| # | 機能 | 説明 | GitHub API |
|---|------|------|------------|
| F1 | タスク一覧表示 | OPENなIssueを優先度タブで分類表示 | `GET /repos/{owner}/{repo}/issues` |
| F2 | タブ切り替え | 🔴緊急 / 🟡今週 / 🔵次週以降 / 👀確認待ち の4タブ | ラベルでフィルタリング |
| F3 | カテゴリフィルター | ドロップダウンでカテゴリラベル絞り込み | クライアント側フィルター |
| F4 | ソート | 優先度順 / カテゴリ順 / 期限順 / 作成日順 | クライアント側ソート |
| F5 | タスク詳細表示 | カードタップでアコーディオン展開、本文・コメント表示 | `GET /repos/{owner}/{repo}/issues/{number}/comments` |
| F6 | メモ追記 | テキスト入力でコメント追加 | `POST /repos/{owner}/{repo}/issues/{number}/comments` |
| F7 | 優先度変更 | ボタンで優先度ラベルを付け替え | `PATCH /repos/{owner}/{repo}/issues/{number}/labels` |
| F8 | タスク完了 | Issue を Close | `PATCH /repos/{owner}/{repo}/issues/{number}` (state: closed) |
| F9 | 新規タスク作成 | タイトル・詳細・優先度・カテゴリ・期限を入力してIssue作成 | `POST /repos/{owner}/{repo}/issues` |
| F10 | GitHubリンク | 元のIssueページへのリンク | URL生成 |

### 期限の扱い

- Issue本文の先頭行に `📅 期限: YYYY-MM-DD` の形式で記載
- 新規作成時に期限入力があれば本文に自動挿入
- 一覧表示時にパースして表示
- 期限3日以内: ⚠️ アラート表示
- 期限超過: 🔥 超過表示（赤色）

### ラベルなしIssue の扱い

- 優先度ラベルなし → 「🟡 今週」タブに表示（デフォルト）
- カテゴリラベルなし → カテゴリバッジ非表示

## 技術構成

| 要素 | 選択 | 理由 |
|------|------|------|
| フレームワーク | React 19 + Vite | 軽量・高速ビルド |
| スタイル | Tailwind CSS v4 | モバイルファーストで素早く構築 |
| 状態管理 | React hooks（useState/useEffect） | 小規模なので十分 |
| Markdown描画 | react-markdown | Issue本文の表示用 |
| API通信 | fetch（ネイティブ） | 追加ライブラリ不要 |
| 認証 | GitHub Personal Access Token | localStorage に保存、初回アクセス時に入力 |
| ホスティング | GitHub Pages | 無料、Vite の静的ビルドをデプロイ |
| レスポンシブ | モバイルファースト | `max-w-md mx-auto` でPC時も縦型維持 |

## 制約・方針

- **サーバーレス**: バックエンド不要。すべてクライアントサイドで完結
- **無料範囲内**: GitHub API（認証済み: 5,000リクエスト/時）、GitHub Pages（無料）
- **横スクロール禁止**: 全コンポーネントを `max-width: 100vw` 以内に収める
- **オフライン非対応**: ネットワーク接続必須（GitHub API依存）
- **シングルユーザー**: 認証・権限管理は不要
